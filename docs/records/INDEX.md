# Documentação — Índice

## Estrutura
- active/: arquivos prioritários e atuais
- archive/duplicates/: duplicados (" 2" no nome)
- reports/: relatórios
- plans/: planos, roadmaps, checklists, tarefas
- guides/: guias e manuais
- audits/: auditorias
- legal/: jurídico e compliance
- marketing/: marca e mercado
- testing/: testes e verificações
- deployment/: deploy e releases
- security/: segurança
- finance/: financeiro
- hangarshare/: materiais específicos de HangarShare (funcionalidade)
- careers/: materiais de carreira
- notifications/: sistema de notificações
- operations/: operação/rotina
- logs/: arquivos .txt e logs
- db/: relatórios de banco de dados (quando movidos)
- misc/: itens diversos
- roles/: documentos por perfil (admin/dev/qa/ops/legal/product)

## Regras
- Não adicionar .md/.pdf/.txt na raiz do projeto.
- Sempre priorizar docs/records/active para documentos atuais.
- Atualizar logbook/ ao final de cada sessão.
